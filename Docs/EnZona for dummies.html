Chequear que esté suscrito el comercio que creamos a la API de pagos y de qr

Por defecto las tiendas nuevas se subscriben, se debe chequear si en la sección de Payment API > tab Consola API > lista de comercios a probar, aparece nuestro comercio 

(opcional) subscribir manualmente en caso de que no esté suscrito el comercio

En la sección Comercios seleccionar el nuestro y dar click en el botón Subscribe.

Generar token de acceso

En el listado de todos los comercios disponibles se selecciona el nuestro del listado > del tab Sandbox/Production Keys se obtienen las llaves de Consumer Key y Consumer Secret.  Con ellas se solicita la generación del token desde el sitio o desde la consola.  Para el caso de la consola (paso 6) se requiere codificar el par de llaves a BASE64, podemos para esto utilizar el sitio BASE64ENCODE codificando la cadena "consumer_key:consumer_secret" (incluyendo el caracter ":" en el medio de ambas llaves).  La cadena ya codificada es la que se utilizaría en el paso 6.

PROBAR las apis

Entrar en la API que queremos probar, en el tab Consola API seleccionamos el comercio que queremos probar y escribimos el token generado en el paso anterior.  Seleccionamos la API que queremos probar, buscándola en el listado y damos click en Try it out.  Garantizamos que los parámetros básicos de esa llamada estén cumplidos (algunas fallan sin esto) y damos click en "Execute".  Tras este paso se devolverá el resultado de la llamada.
